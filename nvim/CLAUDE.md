# Neovim Configuration

A minimal, modern, professional Neovim configuration. Assumes Neovim 0.11+.

## Principles

- Simple and readable — no clever abstractions or over-engineering
- Modern but stable — use current Neovim APIs, avoid deprecated patterns, pin plugin branches/versions where appropriate, prefer mature plugins over bleeding-edge alternatives
- Only popular, well-maintained plugins from reputable authors (folke, tpope, stevearc, nvim-\* orgs, etc.)
- No gimmicks — every plugin must serve a clear, practical purpose
- Minimal dependencies — only add a plugin when it provides significant value over native functionality; evaluate the security risk of each dependency against its benefit
- Proper lazy loading throughout
